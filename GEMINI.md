# news_check プロジェクト規約

本ファイルは、news_check リポジトリ特有のコーディング規約およびワークフローを定めたものです。グローバルな規約（`user_global`）を補完し、本プロジェクトにおいて優先的に適用されます。

## 1. ワークフロー自動化規約

GitHub Issueの作成、コード修正、プッシュ、PR作成の一連の流れにおいて、以下のルールを遵守してください。

### 1.1 PR（Pull Request）の作成と更新
* **変更内容の記載**: PRの説明文（Body）には、直近のプッシュ内容だけでなく、**`main`ブランチとの差分を全て**詳しく記載してください。
* **関連付け**: `gh pr create` を使用してPRを作成する際、説明文に `Close #<Issue番号>` を含めてIssueを関連付けてください。

### 1.2 Terraform ワークフロー
* **認証確認**:
   * AIが `aws sts get-caller-identity` を実行し、AWS CLIが使用可能か確認する。
   * 認証エラーの場合はユーザーに報告し、認証を依頼する。
* **実行計画 (Plan)**:
   * `terraform plan` を実行し、結果を一時ファイルに出力してユーザーに提示する。
* **適用結果の確認**:
   * `terraform apply` 実行後、正常終了した場合は次ステップへ、エラー時は詳細を報告する。

## 2. 開発規約
* **ブランチ命名**: `feature/issue-<Issue番号>-<タイトル>` または `fix/issue-<Issue番号>-<タイトル>` の形式を推奨します。
* **コミットメッセージ**: Conventional Commits に従い、日本語で詳細を記述してください。
