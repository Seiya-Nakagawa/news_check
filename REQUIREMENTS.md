# 要件定義書: News Check Service

## 1. 概要

本プロジェクトは、ユーザーの毎日のニュースチェック作業を自動化するWebサービスを作成することを目的とします。YouTubeでニュース動画を検索し、AIを使用して要約を行い、その結果をユーザーに提示します。

## 2. 背景・課題

現代において、日々のニュース収集は重要ですが、以下の課題を抱えています。

1. **収集にかかる時間**:

    - テレビ・動画等で確認する場合は、視聴するのに時間がかかる。
    - ニュース記事を読む場合は、1つの記事に対して1ページとなるため、複数の記事を読むのにページ検索・遷移する必要があり時間がかかる。
    - 新聞もページ数が多いため、読むのに時間がかかる。

2. **情報の取捨選択**: 膨大な情報の中から、自分にとって重要なニュースを見極めることが難しく、「どの程度チェックすればよいか」の判断に迷うことが多い。

3. **効率性の欠如**: 動画ニュースなどは詳細な情報が得られる反面、視聴に時間がかかり、短時間での概要把握には不向きである。

本サービスは、YouTube上のニュース動画をAIで要約・構造化することで、これらの課題を解決し、短時間で満足度の高いニュースチェック体験を提供することを目指します。

## 3. 機能要件

### 3.1. YouTube動画検索・選定

- **FR-01**: システムは「ニュース」というキーワード（または特定のニュースチャンネル）でYouTubeを検索する。
- **FR-02**: 以下の条件で動画をフィルタリングする。
  - **最新**: 過去24時間以内（または適切な期間）にアップロードされたもの。
  - **非ライブ**: 現在ライブ配信中のものを除外（アーカイブ/VODコンテンツを対象）。
- **FR-03**: システムは最適な動画を自動選択する、またはユーザーがリストから選択できるようにする。

### 3.2. AI要約

- **FR-04**: システムは選択された動画からコンテンツ（字幕/音声）を抽出する。
- **FR-05**: システムは動画内で議論されているニュースのトピックについて、構造化された要約を生成する。

### 3.3. ユーザーインターフェース

- **FR-06**: システムは以下の機能を持つWebインターフェースを提供する。
  - ニュースチェックの実行。
  - 選択された動画の視聴（埋め込みプレーヤー）。
  - AI生成要約の閲覧。
  - (オプション) 元動画へのリンク。

### 3.4. 自動実行・配信

- **FR-07**: システムは設定されたスケジュール（例: 朝・昼・夜）に基づき、自動的にニュースチェックと要約生成プロセスを実行する。
- **FR-08**: 生成された要約は、ユーザーが後で確認できるように保存または通知される。

## 4. 非機能要件

- **NFR-01**: **ユーザビリティ**: UIはシンプルで清潔感があり、「プレミアム」なデザイン（モダンで洗練されたデザインを採用）とする。
- **NFR-02**: **パフォーマンス**: 要約生成は妥当な時間内（例: 選定後1分以内）に完了すること。
- **NFR-03**: **可用性**: 複数ユーザーによる利用または一般公開を想定しているが、ユーザに大きな影響を与えるシステムではないため、コストに影響のある可用性は考慮しない。

## 5. 技術的制約・前提

- **YouTubeデータ**: YouTube Data API を使用する（クォータ制限等を考慮）。
- **AIモデル**: 長文コンテキスト（動画の字幕/音声）を効果的に処理するため、Gemini API (Google AI Studio API経由) の適切なモデルを使用する。
